## Phase 2 — Canyon geometry + boundary conditions ✅
- **Config-driven geometry** (`geometry.build_canyon`): building height H, street
  width W = H/AR, building width, COST 732 margins -> solid mask + index sets
  (street, buildings, roof row, cavity, source row). Tested across aspect ratios.
- **Boundary conditions** (all wired into `solver.CanyonSimulation`):
  - bounce-back walls (ground + buildings);
  - free-slip top (specular via post-collision f) -- validated independently
    against the half-channel analytic profile (zero penetration, zero top shear);
  - non-equilibrium-extrapolation velocity inlet (log-law profile) + constant-
    pressure outlet -- replaced Zou/He, which seeded a corner instability;
  - **outlet viscosity sponge** to absorb the reflective-BC acoustic standing
    wave (a spatially-varying tau field);
  - inlet **ramp** + rest start to avoid the startup shock.
- **Unsteadiness handled by time-averaging**: the canyon flow sheds vortices
  above Re ~ O(100), so `run()` reports the time-averaged mean field (the
  machinery the production LES runs will reuse). Low Re (<~70) is steady.
- **H/W = 1 demonstration** (`configs/canyon_demo.yaml`, Re = 100, BGK + sponge,
  time-averaged over 60k steps): a clean **single clockwise vortex** —
  cavity circulation = -0.078 (clockwise), one centreline u_x reversal, up the
  windward wall / down the leeward wall, downstream aloft / return flow at the
  floor; **mass imbalance = 0.15%** (inflow ≈ outflow). Figure
  `figures/canyon_flow_HW1.{png,svg}` from `scripts/make_figures.py`.
- **Tests:** 27 pass (added geometry, free-slip, and canyon stability/mass/
  single-vortex/config-driven tests).
